# DocViewerApp

## Dev сервер
Для запуска локального dev-сервера выполните
```bash
ng serve
```

## Использование приложения

### Начальная страница [/viewer/1](http://localhost:4200/viewer/1)

### Работа с аннотациями
1) Для добавления анотаций необходимо зажать курсор мыши и потянуть в сторону.  
2) Для удаления - нажать на соответствующую кнопоку аннотации.  
3) Для перемещения анотации нужно зажать курсор в люом месте аннотации и перетащить в нужное место.
4) Для вывода данных документа с аннотациями нужно нажать на кнопку "Save" и посмотреть результат в консоли.

## Проблемы и особенности реализации
Я решил пойти путём масштабирования всего документа через изменение размера контейнера
#### Плюсы:
- Максимальная гибкость решения - можно управлять какие части содержимого страницы
и самих аннотаций будут масштабироваться, а какие оставляем в естественном размере
(к примеру, кнопки управления аннотацией и прочие возможные элементы управления страницей)
#### Минусы:
- Необходимость учитывать и масштабировать все надлежащие части аннотации
(к прим., положение, размеры, размер текста и т.д.)
- Необходимость перевеодить и поддежривать координаты в относительных единицах (%)

### Альтеранивные пути решения и причины отказа от них
1) CSS scale() - самый очевидный и простой вариант, но
   - при масштабировании невозможно корректно центрировать такой элемент
   - при масштабировании будет расползаться весь слой вместе с эелементами управления
   - по данным MDN scale - нагружен для анимаций
2) SVG-аннотации - более экзотический метод, благодаря viewPort позволяет не задумываться о масштабировании вовсе, но
   - при масштабировании будет расползаться весь слой вместе с эелементами управления
   - ограниченность svg в плане удобства и целесообразности использования

### Недоработки
- В решении не реализована возможность рисовать аннотацию в отрицательную сторону.
Необходимо учитывать и обрабатывать отрицательные координаты.
- Размер шрифта аннотаций не меняется при масштабировании
